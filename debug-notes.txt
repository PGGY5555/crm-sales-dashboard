Excel Import Bug Analysis - Job #3 stuck at pending
=================================================

Root cause analysis:
1. Job #3 (2021_20260308.xlsx) created at 12:03:12 UTC, status=pending, processedRows=0
2. The setImmediate background task never started processing
3. Possible reasons:
   a. Production deployment platform may kill the process after response is sent
   b. The countExcelRows() call in background is synchronous and blocks the event loop
   c. Memory pressure from parsing 6MB+ Excel in memory

Performance issues:
1. importCustomersFromExcel does individual INSERT per row (line 343)
   - 61,813 rows = 61,813 individual SQL queries
   - Each with onDuplicateKeyUpdate
2. Job #2 took 72 minutes for 37,573 rows = ~8.7 rows/second (extremely slow)

Fix plan:
1. Use MySQL batch INSERT with VALUES (...), (...), ... for true bulk insert
2. Add a tRPC endpoint to retry/restart stuck jobs
3. Add better error handling and logging in background task
4. Increase BATCH_SIZE from 200 to 500 for bulk inserts
5. Use raw SQL for bulk upsert instead of ORM per-row
